# Video Collector v3 (Telethon + Aiogram)

Крутой “боевой” бот с логированием и понятным прогрессом.

## Что умеет
- Сканирует группы, супергруппы и каналы аккаунта (через MTProto Telethon), но пропускает лички, Избранное, `Подборки 18+` и каналы/группы, название которых начинается с `Vertical`.
- Ищет подходящие видео, **скачивает каждое на сервер**, отправляет файлом в @Content_Vertical_BOT и удаляет локальную копию перед переходом к следующему видео.
- По каждому выбранному каналу/группе проходит без лимита количества сообщений; режимы периода (`day`/`week`/`month`) всё равно ограничивают поиск датой.
- Управление через control-bot (Bot API): постоянное меню Telegram под полем ввода, кнопка скрытия меню, обработка присланных ссылок на страницы с видео, выбор чатов, быстрый выбор всей страницы, обновление списка, период, порядок, отчёты, статистика ссылок, стоп.
- Показ прогресса: постоянное уведомление по ручной очереди ссылок, сколько осталось в очереди, текущий этап загрузки, разрешение/размер/длительность скачанного видео, статус превью, сколько чатов обработано, сколько сообщений проверено, сколько видео найдено/подошло и сколько загружено в целевой бот.
- Логи в консоль и файл `logs/app.log` (ротируются).
- Отчёт после сканирования показывает причины отсева видео и топ чатов по найденным роликам; отдельная кнопка `🔗 Ссылки` ведёт статистику ручных загрузок по ссылкам и последние ошибки.


## Режимы сканирования
- `new` — только сообщения новее последней сохранённой позиции по каждому чату.
- `day` — сообщения за последние 24 часа.
- `week` — сообщения за последние 7 дней.
- `month` — сообщения за последние 30 дней.
- `all` — вся история доступных групп/каналов. Только этот режим для всех чатов обновляет список “пустых” каналов/групп.

## Новые удобства меню
- Основные действия закреплены в меню Telegram под полем ввода, а inline-меню остаётся на экранах статуса и отчётов. Кнопка `🙈 Скрыть меню` убирает клавиатуру, `/start` возвращает её обратно.
- В выборе чатов есть кнопки “Выбрать страницу”, “Снять страницу” и “Обновить список”.
- Доступны команды `/start`, `/status` и `/help` для быстрого доступа без кнопок.
- Отчёт показывает найденные видео, причины отсева (`не вертикальные`, `короткие`, `узкие`, `маленький размер`) и топ чатов по результативности.
- Действия удаления/выхода из пустых каналов можно безопасно проверить с `DRY_RUN_DELETE=true`.

## Фильтр видео
- Вертикальное: h > w
- Длительность: >= 180 секунд
- Ширина: w >= 900
- Размер: >= 10 МБ на минуту (пропорционально)


## Linux / systemd
```bash
cd /home/botuser/video_collector
cp .env.example .env  # если файла .env ещё нет, затем заполни токены
./run.sh
```

`run.sh` сам создаёт `venv`, проверяет наличие обязательных Python-модулей (включая `yt_dlp` и `imageio_ffmpeg`), устанавливает зависимости из `requirements.txt` и запускает `main.py`. Если виртуальное окружение уже было создано, но в нём не хватает пакета, `run.sh` переустановит зависимости даже при неизменном `requirements.txt`. Дополнительно при ручной загрузке внешней ссылки бот сам попробует установить `yt-dlp` в тот Python, которым сейчас запущен сервис, чтобы очередь не падала только из-за запуска не через `run.sh`. Если автоматическая установка не прошла (например, нет прав или сети), в ошибке будет показана команда `pip`, которую нужно выполнить вручную.

Для systemd используй `video_collector.service`: он ожидает проект в `%h/video_collector` и стартует `%h/video_collector/run.sh`. После обновления unit-файла выполни:
```bash
sudo systemctl daemon-reload
sudo systemctl restart video_collector
sudo journalctl -u video_collector -n 100 -f
```

## Ручная очередь по ссылкам
Просто пришли control-bot одну или несколько ссылок на страницы с видео (например YouTube/TikTok/Instagram/другие сайты, поддерживаемые `yt-dlp`; Telegram-ссылки тоже остаются доступными). Бот сам найдёт видео на странице, выберет лучшее доступное качество (`bestvideo*+bestaudio/best` при наличии системного `ffmpeg` или bundled `imageio-ffmpeg`, иначе `best[ext=mp4]/best`), скачает ролик во временную папку на сервере, отправит файлом в @Content_Vertical_BOT, дождётся окончания загрузки и удалит локальный файл. Ссылки обрабатываются строго по очереди: если прислать новые ссылки во время скачивания/загрузки текущего ролика, бот добавит их в конец активной очереди и сразу покажет позиции. Пока очередь работает, новое сканирование не запускается.

### Важные сценарии и ограничения
- Для лучшего качества и обложек при отправке нужен `ffmpeg`: бот сначала ищет системный бинарник, затем использует bundled `imageio-ffmpeg` из Python-зависимостей. Многие сайты отдают видео и звук отдельными потоками, а `yt-dlp` объединяет их в итоговый MP4. Если оба варианта `ffmpeg` недоступны, бот не прерывает очередь и скачивает лучший доступный единый файл (`best[ext=mp4]/best`), но качество может быть ниже, а обложка может не прикрепиться.
- Если сайт требует авторизацию, возрастное подтверждение, cookies или защищённую сессию, укажи файл cookies через `YTDLP_COOKIES_FILE`.
- Плейлисты и страницы с несколькими роликами ограничены первым найденным/основным видео (`noplaylist` + `playlist_items=1`), чтобы случайно не скачать десятки файлов.
- Перед загрузкой проверяется свободное место (`MIN_FREE_DISK_MB`, по умолчанию 1024 МБ), при нехватке места бот сначала удаляет старые временные папки скачивания, а затем применяет лимит размера отправки в Telegram (`MAX_UPLOAD_SIZE_MB`); слишком большой файл будет удалён и попадёт в ошибку очереди.
- Если `yt-dlp` не поддерживает конкретный сайт, сайт отдаёт DRM/стрим без файла или блокирует сервер по IP, ссылка будет отмечена ошибкой, а очередь продолжит следующую ссылку.

## Конфигурация
Скопируй `.env.example` в `.env` и заполни токены/ключи. Локальный `.env`, сессии, базы и логи добавлены в `.gitignore`, чтобы не утекали секреты и runtime-состояние.

## Обложки видео
Бот генерирует JPEG-обложку через системный `ffmpeg` или пакет `imageio-ffmpeg`, пробует несколько моментов видео и несколько размеров до лимита Telegram для кастомной миниатюры. Если обложка всё равно не появилась, проверь логи `Generated thumbnail` / `Cannot generate video thumbnail without ffmpeg` и что сервис запущен через актуальное виртуальное окружение из `run.sh`.

## QA: полный сценарий проверки бота
1. Запусти сервис через `./run.sh` и убедись, что в логах есть авторизация Telethon и нет ошибок `BOT_TOKEN`, `API_ID/API_HASH`, `PHONE`.
2. В Telegram отправь `/start`: должны появиться кнопки `🔎 Все чаты`, `📌 Выбрать`, `⏱ Период`, `🔁 Порядок`, `📊 Отчёты`, `🔗 Ссылки`, `🧹 Пустые`, `🚫 Исключения`, `📄 Статус`, `ℹ️ Помощь`, `🙈 Скрыть меню`.
3. Нажми `🙈 Скрыть меню`: клавиатура должна исчезнуть; отправь `/start`, чтобы вернуть её.
4. Отправь одну ссылку на видео. Бот должен создать постоянное сообщение статуса, показать этапы поиска/скачивания/превью/загрузки, разрешение, размер, длительность и итог отправки в `TARGET_BOT_USERNAME`.
5. Пока первая ссылка обрабатывается, отправь ещё ссылки. Они должны добавиться в конец активной очереди, а статус должен показать новые позиции и остаток.
6. После завершения открой `🔗 Ссылки`: должны быть видны общее количество задач, успешные/ошибочные загрузки, суммарный размер, длительность и последние ошибки.
7. Для сканирования выбери период и порядок, затем запусти `🔎 Все чаты` или `📌 Выбрать` → `🚀 Старт`. В прогрессе должны меняться чат, счётчики, этапы, размер/разрешение найденного видео и статус превью.
8. Нажми `⛔ Стоп` во время долгого скана: бот должен завершить текущий безопасный участок, показать отчёт со статусом остановки и вернуть обычное меню.
9. Открой `📊 Отчёты`, `🧹 Пустые`, `🚫 Исключения` и проверь навигацию, исключение, возврат и подтверждение удаления. Для безопасной проверки удаления используй `DRY_RUN_DELETE=true`.
