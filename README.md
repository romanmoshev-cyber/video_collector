# Video Collector v3 (Telethon + Aiogram)

Крутой “боевой” бот с логированием и понятным прогрессом.

## Что умеет
- Сканирует группы, супергруппы и каналы аккаунта (через MTProto Telethon), но пропускает лички, Избранное, `Подборки 18+` и каналы/группы, название которых начинается с `Vertical`.
- Ищет подходящие видео, **скачивает каждое на сервер**, отправляет файлом в @Content_Vertical_BOT и удаляет локальную копию перед переходом к следующему видео.
- По каждому выбранному каналу/группе проходит без лимита количества сообщений; режимы периода (`day`/`week`/`month`) всё равно ограничивают поиск датой.
- Управление через control-bot (Bot API): меню, обработка присланных ссылок на страницы с видео, выбор чатов, быстрый выбор всей страницы, обновление списка, период, порядок, стоп.
- Показ прогресса: сколько чатов обработано, сколько сообщений проверено, сколько видео найдено/подошло и сколько загружено в целевой бот.
- Логи в консоль и файл `logs/app.log` (ротируются).
- Отчёт после сканирования показывает причины отсева видео и топ чатов по найденным роликам.


## Режимы сканирования
- `new` — только сообщения новее последней сохранённой позиции по каждому чату.
- `day` — сообщения за последние 24 часа.
- `week` — сообщения за последние 7 дней.
- `month` — сообщения за последние 30 дней.
- `all` — вся история доступных групп/каналов. Только этот режим для всех чатов обновляет список “пустых” каналов/групп.

## Новые удобства меню
- В выборе чатов есть кнопки “Выбрать страницу”, “Снять страницу” и “Обновить список”.
- Доступны команды `/start`, `/status` и `/help` для быстрого доступа без кнопок.
- Отчёт показывает найденные видео, причины отсева (`не вертикальные`, `короткие`, `узкие`, `маленький размер`) и топ чатов по результативности.
- Действия удаления/выхода из пустых каналов можно безопасно проверить с `DRY_RUN_DELETE=true`.

## Фильтр видео
- Вертикальное: h > w
- Длительность: >= 180 секунд
- Ширина: w >= 900
- Размер: >= 10 МБ на минуту (пропорционально)


## Linux / systemd
```bash
cd /home/botuser/video_collector
cp .env.example .env  # если файла .env ещё нет, затем заполни токены
./run.sh
```

`run.sh` сам создаёт `venv`, проверяет наличие обязательных Python-модулей (включая `yt_dlp`), устанавливает зависимости из `requirements.txt` и запускает `main.py`. Если виртуальное окружение уже было создано, но в нём не хватает пакета, `run.sh` переустановит зависимости даже при неизменном `requirements.txt`. Дополнительно при ручной загрузке внешней ссылки бот сам попробует установить `yt-dlp` в тот Python, которым сейчас запущен сервис, чтобы очередь не падала только из-за запуска не через `run.sh`. Если автоматическая установка не прошла (например, нет прав или сети), в ошибке будет показана команда `pip`, которую нужно выполнить вручную.

Для systemd используй `video_collector.service`: он ожидает проект в `%h/video_collector` и стартует `%h/video_collector/run.sh`. После обновления unit-файла выполни:
```bash
sudo systemctl daemon-reload
sudo systemctl restart video_collector
sudo journalctl -u video_collector -n 100 -f
```

## Ручная очередь по ссылкам
Просто пришли control-bot одну или несколько ссылок на страницы с видео (например YouTube/TikTok/Instagram/другие сайты, поддерживаемые `yt-dlp`; Telegram-ссылки тоже остаются доступными). Бот сам найдёт видео на странице, выберет лучшее доступное качество (`bestvideo*+bestaudio/best`), скачает ролик во временную папку на сервере, отправит файлом в @Content_Vertical_BOT, дождётся окончания загрузки и удалит локальный файл. Ссылки обрабатываются строго по очереди, поэтому пока очередь работает, новое сканирование не запускается.

### Важные сценарии и ограничения
- Для лучшего качества нужен установленный `ffmpeg`: многие сайты отдают видео и звук отдельными потоками, а `yt-dlp` объединяет их в итоговый MP4.
- Если сайт требует авторизацию, возрастное подтверждение, cookies или защищённую сессию, укажи файл cookies через `YTDLP_COOKIES_FILE`.
- Плейлисты и страницы с несколькими роликами ограничены первым найденным/основным видео (`noplaylist` + `playlist_items=1`), чтобы случайно не скачать десятки файлов.
- Перед загрузкой проверяется свободное место (`MIN_FREE_DISK_MB`) и лимит размера отправки в Telegram (`MAX_UPLOAD_SIZE_MB`); слишком большой файл будет удалён и попадёт в ошибку очереди.
- Если `yt-dlp` не поддерживает конкретный сайт, сайт отдаёт DRM/стрим без файла или блокирует сервер по IP, ссылка будет отмечена ошибкой, а очередь продолжит следующую ссылку.

## Конфигурация
Скопируй `.env.example` в `.env` и заполни токены/ключи. Локальный `.env`, сессии, базы и логи добавлены в `.gitignore`, чтобы не утекали секреты и runtime-состояние.
