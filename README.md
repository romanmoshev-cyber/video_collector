# Forward Bot (Telethon + Aiogram)

Простой управляемый бот для пересылки сообщений из Telegram-каналов и групп в целевой бот или чат.

## Что осталось
- Источники: только каналы, группы и супергруппы аккаунта Telethon.
- Пересылка: сообщения пересылаются напрямую через `forward_messages`, без скачивания файлов и без видео-фильтров.
- Защита от дублей: уже пересланные сообщения сохраняются в SQLite и повторно не отправляются.
- Управление через Bot API: нижнее меню Telegram, выбор источников, запуск по всем источникам, период, порядок, статус и мягкая остановка.
- Логи в консоль и файл `logs/app.log`.
- Heartbeat и watchdog для контроля зависаний.

## Режимы пересылки
- `new` — только сообщения новее последней сохранённой позиции по каждому источнику.
- `day` — сообщения за последние 24 часа.
- `week` — сообщения за последние 7 дней.
- `month` — сообщения за последние 30 дней.
- `all` — вся доступная история выбранных каналов и групп.

## Меню управления
- `🔎 Все` — переслать из всех доступных каналов и групп.
- `📌 Выбрать` — отметить конкретные источники и запустить только их.
- `⏱ Период` — выбрать глубину истории.
- `🔁 Порядок` — новые → старые или старые → новые.
- `📄 Статус` — текущий процесс, heartbeat и последний итог.
- `⛔ Стоп` — мягко остановить текущую пересылку.

Доступны команды `/start`, `/status` и `/help`.

## Установка
```powershell
python -m venv venv
.\venv\Scripts\Activate.ps1
pip install -r requirements.txt
copy .env.example .env
python main.py
```

## Конфигурация
Скопируй `.env.example` в `.env` и заполни значения:

- `BOT_TOKEN` — токен управляющего Bot API бота.
- `API_ID` / `API_HASH` — MTProto параметры аккаунта Telegram.
- `PHONE` — телефон аккаунта для первичной авторизации Telethon.
- `TG_2FA_PASSWORD` — пароль 2FA, если включён.
- `TARGET_BOT_USERNAME` — username целевого бота или чата для пересылки, без `@` или с ним.
- `ALLOWED_USERS` — необязательный CSV-список Telegram user id, которым разрешено управлять ботом.
- `EXCLUDED_CHAT_IDS` — необязательный CSV-список id каналов/групп, которые нужно пропустить.

Локальный `.env`, сессии, базы и логи должны оставаться вне git.

## Логи
- `logs/app.log`
- смотреть “хвост” в PowerShell:
```powershell
Get-Content .\logs\app.log -Tail 200 -Wait
```

## Важно
Перед первым запуском открой целевой бот/чат с аккаунта Telethon и убедись, что аккаунту разрешено туда отправлять сообщения.
